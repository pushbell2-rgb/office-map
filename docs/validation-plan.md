## 1. 핵심 가설

> **"3D 인터랙티브 도면과 실시간 위치 공유를 사용하면, 회의실 탐색 시간을 80% 이상 단축하고 동료 위치 확인을 위한 Slack 메시지를 50% 이상 줄일 수 있다."**

### 베이스라인 수립 근거

기존 불편 수치는 팀 내 설문(n=8, 2026-03-10 실시)과 Slack 채널 로그 분석을 통해 도출:

| 행동 | 현재 소요 시간/횟수 | 측정 근거 |
|------|-------------------|---------|
| 회의실 위치 파악 | 평균 6.2분 (직접 탐색 or 동료 질문) | 팀 내 설문 n=8, 5점 척도 자기 보고 환산 |
| 동료 위치 확인 | Slack 발송 후 평균 4~8분 대기 | \#general 채널 14일간 "어디야" 스레드 21건 분석 |
| 신규 입사자 적응 | 회의실 체계 파악에 평균 1~2주 | 온보딩 담당자 인터뷰 2건 |

### 목표 상태

| 행동 | 목표 소요 시간 | 단축률 |
|------|-------------|--------|
| 회의실 위치 파악 | **30초 이내** (검색 → fly-to 확인) | 92%+ 단축 |
| 동료 위치 확인 | **즉시** (지도에서 핀 확인) | Slack 메시지 50%+ 감소 |

## 3. 검증 실행 계획

### Step 1 — 기능 체크리스트 (배포 전, 개발자 단독)

**실행 방법**: 로컬(`http://localhost:3001`)에서 체크리스트 순서대로 직접 실행

```
[ ] 3D 도면이 map2.jpg와 정렬되어 표시됨
[ ] 44개 회의실 박스가 도면 벽 라인에 맞게 배치됨
[ ] 타입별 색상 구분 및 범례 표시 정상
[ ] 검색창 입력 시 실시간 필터링 동작
[ ] 타입 필터 버튼 클릭 시 해당 타입만 표시됨
[ ] 검색 결과 클릭 → fly-to 애니메이션 동작
[ ] 3D 박스 클릭 → 정보 패널 → 닫기 동작
[ ] "내 위치 설정" → 도면 클릭 → 핀 생성
[ ] 브라우저 탭 2개: 한 쪽 위치 설정 → 다른 쪽 즉시 반영 (스톱워치로 1초 확인)
[ ] 한 탭 닫으면 다른 탭에서 해당 핀 사라짐
[ ] T키 → 채팅 입력 → 말풍선 5초 후 자동 삭제
[ ] 방향키로 핀 이동 동작
[ ] 온보딩 가이드 최초 접속 시 표시, 재방문 시 미표시
[ ] ?room=M-7 URL 접속 시 해당 회의실 하이라이트 이동
[ ] 이름 없이 입장 시 "익명" 처리
[ ] 브라우저 콘솔 에러 없음
```

**예상 소요 시간**: 25분

### Step 3 — 동기화 지연 측정

**실행 방법**
1. 동일 PC에서 Chrome 탭 A, B 각각 접속
