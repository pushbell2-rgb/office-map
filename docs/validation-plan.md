# 검증 계획

> 작성일: 2026-03-13 | 서비스: 서울숲 사옥 3D 회의실 찾기 & 실시간 위치 공유

---

## 1. 핵심 가설

> **"3D 인터랙티브 도면과 실시간 위치 공유를 사용하면, 회의실 탐색 시간을 80% 이상 단축하고 동료 위치 확인을 위한 Slack 메시지를 50% 이상 줄일 수 있다."**

### 세부 가설

| ID | 가설 | 검증 방법 |
|----|------|---------|
| H1 | 검색창으로 회의실을 5초 이내에 찾을 수 있다 | 사용성 테스트 (스톱워치) |
| H2 | 별도 설명 없이도 신규 사용자가 3분 내 위치 설정까지 완료한다 | 온보딩 없는 세션 관찰 |
| H3 | 동료 핀 확인 후 Slack 없이 바로 찾아갈 수 있다 | 직접 사용 시나리오 |
| H4 | 20명 동시 접속 시 위치 동기화 지연이 1초 미만이다 | 멀티 탭 부하 테스트 |

## 2. 성공 지표 (KPI)

| 지표 | 목표값 | 측정 방법 | 우선순위 |
|------|--------|---------|---------|
| 회의실 탐색 시간 | 검색 → 위치 파악 **5초 이내** | 사용성 테스트 스톱워치 | P0 |
| 위치 동기화 지연 | 핀 설정 → 다른 화면 반영 **1초 이내** | 2대 브라우저 동시 측정 | P0 |
| 신규 사용자 성공률 | 안내 없이 **1분 내** 위치 설정 완료 **80%** | 세션 관찰 (n=5) | P0 |
| 렌더링 성능 | 데스크톱 Chrome에서 **60fps 유지** | DevTools Performance | P1 |
| 동시 접속 안정성 | **20명** 동시 접속 시 정상 동작 | 로컬 멀티 탭 테스트 | P1 |

## 3. 검증 단계

### Step 1 — 기능 체크리스트 (개발 직후, 혼자 수행)

```
[ ] 3D 도면이 map2.jpg와 정렬되어 표시됨
[ ] 44개 회의실 박스가 도면 벽 라인에 맞게 배치됨
[ ] 타입별 색상 구분 및 범례 정상 표시
[ ] 회의실 이름/ID 검색 시 실시간 필터링 동작
[ ] 검색 결과 클릭 → fly-to 애니메이션 동작
[ ] 3D 박스 클릭 → 정보 패널 표시 → 닫기 동작
[ ] "내 위치 설정" → 도면 클릭 → 핀 생성
[ ] 브라우저 탭 2개: 한 쪽 위치 설정 → 다른 쪽 즉시 반영
[ ] 한 탭 닫으면 다른 탭에서 해당 핀 사라짐
[ ] 이름 없이 입장 시 "익명" 처리
```

### Step 2 — 사용성 테스트 (팀원 대상, n=3~5)

**진행 방식**: URL만 전달, 설명 없이 시작

| 테스트 시나리오 | 성공 기준 |
|--------------|---------|
| "M-7 오타와 회의실을 찾아보세요" | 30초 이내 위치 파악 |
| "동료 핀을 클릭해 그 위치로 이동해보세요" | 1회 시도에 성공 |
| "본인 현재 위치를 설정해보세요" | 설명 없이 1분 내 성공 |
| "방콕 회의실이 어느 타입인지 확인해보세요" | 정보 패널 열어 확인 |

**수집 데이터**:
- 각 태스크 소요 시간
- 막혔던 지점 (관찰)
- 사후 인터뷰: "직관적으로 느껴졌나요?" (5점 척도)

### Step 3 — 부하 테스트 (배포 전)

```
방법: 동일 PC에서 Chrome 탭 10개 동시 열어 각각 위치 설정
측정 항목:
  - 위치 업데이트 후 전체 탭 반영까지 소요 시간
  - CPU 사용률 스파이크 여부
  - 어느 탭에서도 UI 프리즈 없음 확인
성공 기준: 모든 탭에서 1초 이내 동기화, UI 프리즈 없음
```

## 5. 피벗 기준

다음 상황이 확인될 경우 방향 조정:

| 상황 | 조치 |
|------|------|
| 좌표 오차가 방 크기의 30% 이상 | 수동 오프셋 조정 UI 추가 |

| 모바일 프레임 < 15fps | 2D 이미지 + 오버레이 모드 추가 |
| 사용자가 3D 조작을 어려워함 | 줌인/아웃 버튼 UI 추가 |
