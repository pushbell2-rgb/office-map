## 6. 측정 방법 상세 가이드

### 스톱워치 측정 프로토콜

```
1. 테스터에게 URL 공유 (설명 금지)
2. 테스터가 브라우저를 열고 URL을 입력하는 순간 → 스톱워치 시작
3. 시나리오 완료 조건 충족 시 → 스톱워치 중지
4. 완료 조건:
   T1: "이 방 찾았어요"라고 말하거나 카메라가 해당 방으로 이동된 순간
   T2: 본인 핀이 지도에 표시된 순간
   T3: 카메라가 동료 핀 위치로 이동된 순간
   T4: 정보 패널에서 타입을 읽는 순간
```

### Chrome DevTools 동기화 측정 프로토콜

```
1. 탭 A, B 각각 서비스 접속 후 입장
2. 탭 B에서 DevTools → Network → WS 필터
3. socket.io 웹소켓 연결 클릭 → Frames 탭 열기
4. 탭 A에서 "내 위치 설정" 클릭 후 도면 클릭
5. 탭 B Frames에서 "users-update" 수신 프레임 Time 컬럼 확인
6. (탭 A emit 시각 - 탭 B 수신 시각) = 지연
```
