# 검증 계획

> 작성일: 2026-03-13 | 서비스: 서울숲 사옥 3D 회의실 찾기 & 실시간 위치 공유

---

## 1. 핵심 가설

> **"3D 인터랙티브 도면과 실시간 위치 공유를 사용하면, 회의실 탐색 시간을 80% 이상 단축하고 동료 위치 확인을 위한 Slack 메시지를 50% 이상 줄일 수 있다."**

### 베이스라인 (현재 상태)
| 행동 | 현재 소요 시간/횟수 | 측정 근거 |
|------|-------------------|---------|
| 회의실 위치 파악 | 평균 5~8분 (현장 탐색 or 동료에게 질문) | 팀 내 비공식 인터뷰 (n=5) |
| 동료 위치 확인 | Slack 메시지 발송 후 평균 3~10분 대기 | 슬랙 응답 패턴 관찰 |

### 목표 상태
| 행동 | 목표 소요 시간 | 단축률 |
|------|-------------|--------|
| 회의실 위치 파악 | **1분 이내** (검색 → fly-to 확인) | 80%+ 단축 |
| 동료 위치 확인 | **즉시** (지도에서 핀 확인) | Slack 메시지 50%+ 감소 |

---

## 2. 세부 가설 및 검증 방법

| ID | 가설 | 측정 방법 | 성공 기준 | 실패 기준 |
|----|------|---------|---------|---------|
| H1 | 검색창으로 회의실을 1분 이내 찾는다 | 사용성 테스트 스톱워치 | 5명 중 4명이 60초 이내 성공 | 3명 이상 실패 |
| H2 | 안내 없이 신규 사용자가 위치 설정 완료 | 온보딩 없는 세션 관찰 | 5명 중 4명이 3분 이내 성공 | 3명 이상 실패 |
| H3 | 위치 동기화가 1초 이내 반영된다 | 2대 기기 동시 측정 | 10회 중 9회 1초 이내 | 3회 이상 초과 |
| H4 | 20명 동시 접속 시 서비스 안정 유지 | 멀티탭 부하 테스트 | UI 프리즈 없음, 동기화 유지 | 탭 1개 이상 응답 없음 |

## 3. 검증 실행 계획

### Step 1 — 기능 체크리스트 (배포 전, 개발자 단독)

**실행 방법**: 로컬(`http://localhost:3001`)에서 체크리스트 순서대로 직접 실행

```
[ ] 3D 도면이 map2.jpg와 정렬되어 표시됨
[ ] 44개 회의실 박스가 도면 벽 라인에 맞게 배치됨
[ ] 타입별 색상 구분 및 범례 표시 정상
[ ] 검색창 입력 시 실시간 필터링 동작
[ ] 검색 결과 클릭 → fly-to 애니메이션 동작
[ ] 3D 박스 클릭 → 정보 패널 → 닫기 동작
[ ] "내 위치 설정" → 도면 클릭 → 핀 생성
[ ] 브라우저 탭 2개: 한 쪽 위치 설정 → 다른 쪽 즉시 반영 (스톱워치로 1초 확인)
[ ] 한 탭 닫으면 다른 탭에서 해당 핀 사라짐
[ ] 이름 없이 입장 시 "익명" 처리
[ ] 브라우저 콘솔 에러 없음
```

**예상 소요 시간**: 20분

---

### Step 2 — 사용성 테스트 (팀원 대상, n=5)

**준비물**
- 배포 URL (또는 로컬 서버)
- 스톱워치 앱
- 관찰 기록 시트 (아래 양식)

**진행 방법**
1. 테스터에게 URL만 공유, 아무 설명 없이 시작
2. 진행자는 관찰만 하고 힌트 제공 금지
3. 각 시나리오 완료 시 시간 기록

**테스트 시나리오 및 측정 기준**

| # | 시나리오 | 스톱워치 시작 | 성공 조건 | 목표 시간 |
|---|---------|------------|---------|---------|
| T1 | "M-7 오타와 회의실을 찾아보세요" | 화면 진입 시 | 3D에서 위치 확인 | 60초 |
| T2 | "본인 현재 위치를 지도에 표시해보세요" | 입장 직후 | 핀 설치 완료 | 180초 |
| T3 | "동료 핀을 클릭해 그 위치로 이동하세요" | 안내 시작 시 | fly-to 실행 | 30초 |
| T4 | "방콕 회의실이 어떤 타입인지 확인하세요" | 안내 시작 시 | 정보 패널에서 확인 | 45초 |

**관찰 기록 양식**

```
테스터 ID: ___  날짜: ___  장치: ___

T1: [ ]성공 [ ]실패  소요: ___초  막힌 지점: ___
T2: [ ]성공 [ ]실패  소요: ___초  막힌 지점: ___
T3: [ ]성공 [ ]실패  소요: ___초  막힌 지점: ___
T4: [ ]성공 [ ]실패  소요: ___초  막힌 지점: ___

사후 질문 (5점 척도):
Q1. 회의실 위치 파악이 기존보다 빨랐나요?      [ 1  2  3  4  5 ]
Q2. 동료 위치 확인이 직관적이었나요?           [ 1  2  3  4  5 ]
Q3. 전반적으로 사용하기 쉬웠나요?             [ 1  2  3  4  5 ]
```

**예상 소요 시간**: 테스터 1인당 15분

### Step 3 — 동기화 지연 측정

**실행 방법**
1. 기기 A, 기기 B에서 각각 서비스 접속 (또는 동일 PC 탭 2개)
2. 기기 A에서 위치 설정 클릭
3. 기기 B 화면에 핀이 나타나는 시간 스톱워치 측정
4. 10회 반복 → 평균/최대 지연 기록

**결과 기록**
```
회차  지연(초)
 1:   ___
 2:   ___
 ...
10:   ___
평균: ___  최대: ___  성공률(1초이내): ___/10
```

---

### Step 4 — 부하 테스트

**실행 방법**
1. Chrome에서 탭 10개 열어 각각 다른 이름으로 입장
2. 모든 탭에서 동시에 위치 설정
3. 각 탭에서 다른 탭 핀 업데이트 확인

**측정 지표**
- CPU 사용률 (Task Manager)
- 탭별 동기화 지연
- 응답 없음(프리즈) 발생 여부
